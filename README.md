# GitOps Multi-Environment Promotion Pipeline

One app, one repo, promoted **dev → staging → prod** using ArgoCD's
App-of-Apps pattern with Kustomize environment overlays, and a real manual
approval gate before production — using GitHub's actual Environment
protection rules, not a simulated one.

> Portfolio project demonstrating: GitOps (ArgoCD), environment-specific
> config via Kustomize (not copy-pasted YAML), progressive delivery, and a
> genuine two-layer production approval gate.

## What "real" means for this project

Every Kustomize overlay in this repo was built and validated with the actual
`kustomize` binary (v5.4.3) before being committed — not just checked for
YAML syntax. `scripts/validate-overlays.sh` is the exact script that did
that validation, and it also runs on every PR via
`.github/workflows/validate-pr.yml`. The sample app (`app/`) was built and
run locally with real HTTP requests against every endpoint. See
`docs/SETUP.md` for exactly what's been tested here versus what needs a
real cluster/GitHub repo to exercise end-to-end (the ArgoCD sync behavior
and the GitHub Environment approval gate both require actual
infrastructure — they're implemented correctly and documented precisely,
but I want to be upfront about what's been run in this sandbox versus what
you'll run yourself).

## The promotion flow

```
push to main
     │
     ▼
CI builds + tests + pushes image, bumps DEV overlay tag automatically
     │
     ▼
ArgoCD (automated sync) deploys to dev namespace
     │
     │  <- human runs "Promote to Staging" workflow (lightweight, no gate)
     ▼
STAGING overlay tag bumped to whatever's live in dev
     │
     ▼
ArgoCD (automated sync) deploys to staging namespace
     │
     │  <- human runs "Promote to Production" workflow
     │     -> GitHub Environment "production" requires reviewer approval
     │     -> only after approval does the workflow even run
     ▼
PROD overlay tag bumped to whatever's live in staging
     │
     ▼
ArgoCD prod-app (MANUAL sync only) — a human still has to click Sync
```

Two separate gates before anything reaches prod: a required GitHub reviewer
approval (before the Git commit happens), and a manual ArgoCD sync (before
the commit actually gets applied to the cluster). See
`docs/INTERVIEW_TALKING_POINTS.md` for why both exist.

## Repo layout

```
gitops-promo/
├── app/                       # Real FastAPI app — tested locally, reports its own env+version
├── k8s/
│   ├── base/                  # Shared Deployment/Service/ConfigMap
│   └── overlays/
│       ├── dev/                # 1 replica, auto-deployed on every push
│       ├── staging/             # 2 replicas, promoted manually (light gate)
│       └── prod/                # 3 replicas + HPA + PodDisruptionBudget, promoted via approval gate
├── argocd/
│   ├── root-app-of-apps.yaml   # The one Application you apply by hand
│   ├── apps/                   # dev-app, staging-app, prod-app (auto-generated by App-of-Apps)
│   └── projects/               # AppProject restricting what each env can touch
├── .github/workflows/
│   ├── ci.yml                  # Build, test, push image, auto-promote to dev
│   ├── promote-staging.yml     # Manual trigger, no heavy gate
│   ├── promote-prod.yml        # Manual trigger + GitHub Environment approval
│   └── validate-pr.yml         # Runs validate-overlays.sh on every PR touching k8s/
├── scripts/
│   ├── bump-image-tag.sh       # Updates the ConfigMap APP_VERSION per environment
│   └── validate-overlays.sh    # Real kustomize build validation, used locally and in CI
└── docs/                       # Setup guide, architecture, interview talking points
```

## Quickstart

```bash
# 1. Validate the overlays locally (needs the real kustomize binary)
./scripts/validate-overlays.sh

# 2. See exactly what would deploy to each environment
kustomize build k8s/overlays/dev
kustomize build k8s/overlays/staging
kustomize build k8s/overlays/prod

# 3. Build and run the app locally
cd app && pip install -r requirements.txt
ENVIRONMENT=local APP_VERSION=v1 uvicorn main:app --port 8000
curl localhost:8000/
```
